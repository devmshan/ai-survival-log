# Context Graph vs Knowledge Graph

> 원본 URL: https://atlan.com/know/context-graph-vs-knowledge-graph/
> 수집일: 2026-04-13

## 핵심 정의

**Knowledge Graph(지식 그래프)**
엔티티 간의 의미론적 관계를 표현하여 "사물이 무엇인지"를 정의합니다. RDF 트리플 스토어나 프로퍼티 그래프 기반으로 개념적 관계를 모델링합니다.

**Context Graph(컨텍스트 그래프)**
"어떻게 작동하고 왜 결정이 내려졌는지"를 설명하기 위해 지식 그래프에 운영 메타데이터를 추가합니다. 선형성(lineage), 결정 추적, 시간적 맥락, 거버넌스 정책을 포함합니다.

## 주요 차이점 (6가지 차원)

| 구분 | Knowledge Graph | Context Graph |
|------|-----------------|---------------|
| **핵심 목표** | 의미 관계 정의 | AI 운영을 위한 의사결정 정보 |
| **초점** | "무엇인가" - 엔티티, 분류법 | "어떻게 작동하는가" - 라인리지, 정책 |
| **시간 인식** | 정적 또는 천천히 변하는 관계 | 시간 역행 쿼리, 유효기간, 거래 타임스탬프 |
| **데이터 구조** | 노드-엣지-속성 | 노드(정책/결정 포함)-엣지-메타데이터 |
| **쿼리 방식** | SPARQL/Cypher 의미 탐색 | 그래프 탐색 + 운영 필터 |
| **AI 통합** | 의미 이해 제공 | LLM 소비 최적화(토큰 효율, 할루시네이션 감소) |

## Context Graph가 추가하는 4가지 핵심 요소

### 1. 의사결정 추적 및 선례 링크
승인, 예외, 재현 가능한 워크플로우를 캡처하여 결정 경로를 명시적으로 보존합니다.

### 2. 시간적 한정자 및 타임 트래블 지원
유효기간, 거래 타임스탬프, 시간 역행 쿼리를 지원합니다.

### 3. 출처 추적 및 신뢰도 점수
소스 귀속과 품질 신호를 내장 메타데이터로 취급합니다.

### 4. 정책을 그래프 요소로 (거버넌스 노드화)
접근 제어, 데이터 분류, 규정 준수 요구사항을 그래프 노드로 표현합니다.

## AI 할루시네이션 감소

그래프 기반 RAG는 기존 방식 대비 40% 이상의 할루시네이션 감소율을 달성합니다.
