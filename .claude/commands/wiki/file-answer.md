# /wiki:file-answer — 답변을 위키 페이지로 저장

대화에서 생성된 답변이나 분석을 위키 페이지로 저장합니다.
탐색과 질문에서 발견한 지식이 대화 히스토리에 묻히지 않도록 합니다.

## 입력

저장할 내용 (이전 답변 또는 새 텍스트). 인자가 없으면 이전 대화에서 가장 최근의 의미 있는 답변을 사용합니다.

## 워크플로우

### 1단계: 카테고리 결정

사용자에게 질문합니다:

```
이 답변을 어떤 카테고리에 저장할까요?
1. entities/ — 도구, 프레임워크, 인물
2. concepts/ — 아이디어, 패턴, 정의
3. topics/ — 심층 주제 (여러 개념 종합)
4. projects/ — 프로젝트 설계/계획
```

### 2단계: 페이지 생성

- 적절한 파일명 결정 (kebab-case, 영문)
- CLAUDE.md의 frontmatter 스펙에 따라 frontmatter 생성
- 답변 내용을 위키 페이지 형식으로 정리
- 본문에서 관련 위키 페이지를 `[[wikilink]]`로 연결
- `## 관련 페이지` 섹션 추가

### 3단계: 크로스 레퍼런싱

- 관련 기존 페이지의 `## 관련 페이지`에 새 페이지 역링크 추가

### 4단계: 인덱스 및 로그 갱신

- `wiki/index.md`에 새 페이지 추가
- `wiki/log.md`에 file-answer 작업 기록

## 완료 보고

```
페이지 저장 완료:
- 파일: wiki/[category]/[name].md
- 관련 페이지 업데이트: N개
```
