# /journals:backup — 대화 백업

현재 대화 세션의 내용을 `raw/journals/`에 마크다운 백업 파일로 저장합니다.
저장된 파일은 추후 블로그 글 작성의 원천 자료로 재사용하거나 `/wiki:ingest`로 위키에 흡수할 수 있습니다.

## 입력

- **인자 없음**: 대화 내용에서 주제를 자동으로 추론합니다
- **주제 힌트**: `/journals:backup 폴더구조 변경 실행` 처럼 키워드를 넣으면 파일명과 맥락 설명에 반영합니다

## 워크플로우

### 1단계: 메타데이터 결정

다음 정보를 확정합니다:

- **날짜**: 오늘 날짜 (`YYYY-MM-DD`)
- **주제**: 대화 흐름의 핵심 키워드 (인자로 받은 경우 우선 사용)
- **파일명**: `YYYY-MM-DD-{kebab-topic}-conversation-backup.md`
  - 예: `2026-04-18-codex-folder-structure-execution-and-publish-conversation-backup.md`
- **이전 연관 백업**: `raw/journals/`에 같은 날짜 또는 연관 주제로 이미 저장된 파일이 있으면 `참고:` 필드에 명시

### 2단계: 대화 내용 정리

대화 흐름을 다음 기준으로 정리합니다:

- **User 메시지**: 최대한 원문 그대로 인용 (백틱 블록)
- **Assistant 응답**:
  - 짧은 응답은 요지 1-2줄로 압축
  - 실행 결과가 있으면 결과 요약 포함
  - 긴 목록/코드는 "요약 항목 3-5개 + 기타" 형태로 축약
- **턴 수가 많으면** 의미 있는 결정/전환이 있는 턴만 남기고 단순 확인 턴은 병합

### 3단계: 파일 생성

`raw/journals/{파일명}`으로 저장합니다. 형식은 반드시 아래 템플릿을 따릅니다:

```markdown
# 대화 백업: {제목}

- **날짜:** YYYY-MM-DD
- **성격:** 전체 대화 백업
- **맥락:** {한 줄 맥락 — 무엇을 왜 진행한 세션인지}
- **용도:** 추후 블로그 글 작성 시 원천 대화 기록으로 재사용
- **참고:** {연관 이전 백업 파일 경로 — 없으면 생략}

---

## 대화 원문 백업

### User

`{사용자 메시지 원문}`

### Assistant

{응답 요지 또는 실행 결과 요약}

(... 이하 반복 ...)

---

## 세션 핵심 결정

- {이 세션에서 합의/결정된 항목들}

## 이번 세션에서 실제 반영된 항목

- {파일/코드/문서 변경 사항}

## 블로그 글 소재 포인트

- {블로그 글로 발전시킬 수 있는 인사이트}
```

### 4단계: 완료 보고

저장 후 다음을 보고합니다:

```
저장 완료:
- 파일: raw/journals/{파일명}
- 주제: {주제}
- 블로그 소재: {핵심 소재 1-2개}
- 다음 단계: /wiki:ingest 로 위키 흡수 가능
```

## 주의사항

- `raw/journals/`는 불변 계층 — 한 번 저장하면 직접 수정하지 않습니다
- 한 세션에서 여러 파일을 만들어도 되지만, 주제가 명확히 다를 때만 분리합니다
- 같은 날 연속 세션이면 파일명 끝에 `-part2` 등을 붙여 구분합니다
- 보안 정보(API 키, 비밀번호 등)가 대화에 포함된 경우 저장 전 제거합니다
- `/wiki:ingest`로 위키 흡수 시 이 파일을 소스로 사용합니다
