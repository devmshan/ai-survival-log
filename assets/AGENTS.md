# Assets Layer Guide

- `assets/`는 수집 inbox와 채널별 제작 자산 원본 계층을 함께 가진다.
- 아직 채널이 정해지지 않은 수집 자산은 `assets/intake/`에 둔다.
- 블로그용 upstream 이미지는 `assets/blog/`에 둔다.
- 문서 설명은 `docs/`, 실제 자산 원본은 `assets/`에 둔다.
- Web Clipper 템플릿은 `assets/clipper-templates/`에 둔다.
- 이미지/첨부파일 메타 노트 inbox는 `assets/intake/reference-notes/`를 사용한다.
- 이미지 원본 inbox는 `assets/intake/reference-images/`, 첨부파일 inbox는 `assets/intake/attachments/`를 사용한다.
- publish-facing 자산 파일명은 가능하면 ASCII kebab-case를 유지한다.
