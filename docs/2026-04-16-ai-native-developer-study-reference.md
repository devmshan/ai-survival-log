# AI-Native Developer 강의 참고자료

## 용도

이 문서는 블로그 발행용이 아니라, 공부 계획을 세우기 전에 빠르게 훑어보는 내부 참고자료다.

대상 강의:
- `The 10x AI-Native Developer: 회사에서 AI로 압도적 성과를 내는 법`

근거 범위:
- 인프런 공개 소개 페이지와 공개 커리큘럼 섹션명
- 확인일: 2026-04-16

주의:
- 강의 전체 수강 노트가 아니다.
- 공개된 정보만 바탕으로 "어떤 실무 자동화 예시를 다루는 강의처럼 보이는가"를 정리한 메모다.

## 강의에서 읽히는 자동화 예시

- GitHub 협업 자동화: `gh CLI`로 PR, 이슈, 리뷰 흐름을 AI가 대신 다루게 하는 방식
- Jira 태스크 관리 자동화: Jira MCP로 티켓 생성, 정리, 상태 추적 같은 PM성 작업 보조
- 병렬 작업 에이전트 운영: Sub-agents, `claude-squad`로 여러 작업을 동시에 분담
- E2E 테스트 자동화: Playwright MCP로 QA 시나리오 실행과 검증 자동화
- 웹 데이터 수집 자동화: Puppeteer MCP로 반복적인 정보 수집/스크래핑 자동화
- 오픈소스 기여 자동화: 이슈 파악, 코드 수정, PR 작성까지 AI와 함께 처리
- AI 코드 리뷰어 구축: GitHub Actions + CodeRabbit로 24시간 리뷰 자동화
- 인프라 운영 자동화: Terraform으로 AWS 인프라 생성/변경을 코드로 관리
- 모니터링 자동화: 인프라 모니터링 MCP, Langfuse로 시스템/LLM 품질 추적
- 에이전트형 서비스 구현: LangChain Agent로 "생각하고 도구를 쓰는 AI" 형태의 기능 구현

## 한 줄 해석

이 강의에서 말하는 자동화는 `코드 작성 자동화`보다, 개발자가 원래 손으로 하던 `운영/협업/검증/배포` 업무 전체를 AI에게 위임하는 구조에 더 가깝다.

## 회사에서 바로 응용하기 쉬운 형태

- PR 열리면 AI가 변경 요약, 리뷰 포인트, 테스트 누락 체크
- Jira 이슈를 읽고 구현 계획과 작업 순서를 자동 초안 생성
- 회귀 테스트를 AI가 실행하고 실패 원인까지 정리
- 운영 중 장애가 나면 로그와 모니터링 지표를 읽고 1차 분석
- 반복적인 사내 데이터 수집이나 문서 업데이트 작업 자동화

## 회사에서 바로 해볼 수 있는 자동화 프로젝트 10개

### 1. PR 요약 + 리뷰 포인트 생성기

PR이 열리면 변경 파일, 리스크, 테스트 누락 가능성을 AI가 요약해 코멘트로 남긴다.

### 2. Jira 이슈 구현 계획 초안 생성기

Jira 티켓 설명과 관련 레포를 읽고, 구현 순서, 예상 변경 파일, 테스트 항목, 확인 질문을 자동으로 정리한다.

### 3. 회귀 테스트 실행 + 실패 원인 정리 봇

배포 전 또는 PR 단계에서 Playwright나 통합 테스트를 돌리고, 실패 시 원인 후보를 자연어로 정리한다.

### 4. 장애 1차 분석 봇

에러 로그, APM 지표, 최근 배포 이력, 관련 PR을 읽고 가장 가능성 높은 원인과 다음 확인 지점을 제안한다.

### 5. API 문서와 사내 런북 자동 갱신기

코드 변경이나 운영 절차 변경이 생기면 관련 문서 후보를 찾고, 업데이트 초안을 자동 생성한다.

### 6. 반복 운영 쿼리 생성기

운영 중 자주 쓰는 SQL, 로그 검색 쿼리, Kibana/Loki 필터를 템플릿화하고 사건 설명만 넣으면 AI가 즉시 쿼리를 만든다.

### 7. 배포 체크리스트 자동 수행 에이전트

릴리스 노트, 마이그레이션 유무, feature flag 상태, 롤백 포인트를 확인해 배포 전 체크리스트를 자동 수행한다.

### 8. 오픈소스 이슈 대응 보조 워크플로우

외부 오픈소스 이슈를 읽고 재현 절차, 예상 수정 지점, PR 본문 초안까지 AI가 준비한다.

### 9. 경쟁사 또는 기술 동향 모니터링 스크래퍼

특정 문서, 릴리즈 노트, 채용 공고, 제품 업데이트를 주기적으로 수집하고 변화만 요약한다.

### 10. 사내 백오피스 반복 작업 자동화

주간 보고서, 장애 회고 초안, 스프린트 정리, 이슈 상태 점검 같은 반복 작업을 AI가 먼저 정리하고 사람은 검토만 한다.

## 백엔드 개발자 기준 ROI 높은 자동화 3개

### 1. PR 요약 + 리뷰 포인트 생성기

도입 비용이 낮고, 코드 리뷰 시간을 즉시 줄여준다.

### 2. 장애 1차 분석 봇

온콜 초반 대응 시간을 줄이고 MTTR 개선 효과가 크다.

### 3. Jira 이슈 구현 계획 초안 생성기

개발 시작 전의 애매한 정리 작업을 줄이고, 테스트 항목까지 구조화해준다.
