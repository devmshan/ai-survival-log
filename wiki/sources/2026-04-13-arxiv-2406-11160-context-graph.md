# Context Graph 논문 (arXiv:2406.11160)

> 원본: https://arxiv.org/abs/2406.11160
> 저자: Chengjin Xu, Muzhi Li, Cehao Yang, Xuhui Jiang, Lumingyuan Tang, Yiyan Qi, Jian Guo
> 제출: 2024-06-17 | 라이선스: CC BY-NC-ND 4.0

## 핵심 요약

기존 triple 기반 KG의 맥락 정보 부재 문제를 해결하기 위해 **Context Graph(CG)** 개념과 **CGR³ 패러다임** (Retrieve-Rank-Reason)을 제안. KGC와 KGQA 벤치마크에서 SOTA 대비 대폭 성능 향상 달성.

## 주요 기여

1. 삼중항 KG 한계 분석 → `(h, r, t)` 형식은 시간·위치·출처 정보를 잃음
2. Context Graph 정의: `(h, r, t, rc)` — rc는 관계 맥락(relation context)
3. CGR³: LLM을 Retrieve-Rank-Reason 3단계에 통합한 새 패러다임
4. 장꼬리 엔티티(long-tail entity)에서도 강건한 성능

## CGR³ 상세 구조

### Retrieve (검색)
| 방식 | 내용 |
|------|------|
| Supporting Triple | (h, r, ?)에서 h, r 공유 삼중항 검색 |
| Textual Context | Wikidata 라벨 + Wikipedia 문장으로 자연어화 |
| KG Embedding | 학습된 임베딩으로 후보 사전 필터링 |
| LLM Generation | 자연어 질문 변환 후 LLM으로 후보 생성 |

### Rank (순위 결정)
- Llama-3-8B-Instruct + LoRA(rank=16) SFT
- 질문-삼중항-맥락 정합성 평가 → 재정렬

### Reason (추론)
- GPT-3.5-turbo-0125로 최종 추론
- 정보 충분성 판단 후 부족하면 Retrieve 단계 반복
- 최대 깊이 D_max=3, 빔 너비 M=3

## 실험 결과

### KGC (지식그래프 완성)

| 베이스라인 | FB15k-237 Hits@1 향상 | YAGO3-10 Hits@1 향상 |
|-----------|----------------------|---------------------|
| ComplEx | +66.46% | +30.77% |
| RotatE | +21.58% | +10.20% |
| GIE | +11.07% | +3.37% |

### KGQA (지식그래프 질의응답)

| 방법 | QALD10-en | WWQ |
|------|-----------|-----|
| CGR³ | **54.7%** | **78.8%** |
| CGR³ w/o Context | 38.1% | 67.3% |
| 이전 SOTA | 50.2% | 72.6% |

**맥락 유무 차이**: QALD10-en +16.6%p, WWQ +11.5%p

## 핵심 발견

- **맥락(Context)이 결정적**: 순위 결정 단계에서 맥락 제거 시 최대 25% 하락
- **SFT 필수**: 미세조정 없이는 LLM 추론 성능 급감 (15~40%)
- **장꼬리에서 강점**: 이웃이 적은 엔티티에서도 일관된 성능

## 구현 스펙

- GPU: NVIDIA A100-SXM4-40GB × 8
- Optimizer: AdamW, lr=1e-4, LoRA rank=16
- 학습 데이터: FB15k-237, YAGO3-10 (KGC), QALD10-en, WWQ (KGQA)
