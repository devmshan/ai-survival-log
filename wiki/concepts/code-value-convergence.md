# 코드 가치 0 수렴론

## 핵심 주장

**코드 자체의 가치는 0으로 수렴한다.** AI가 코드를 무한히 생성할 수 있게 되면서, 코드는 더 이상 희소 자원이 아니다. 소프트웨어의 중심축이 코드에서 **모델(model)**로 이동하고 있다.

## 소프트웨어의 세 번째 대변혁

소프트웨어 산업은 역사적으로 두 번의 큰 전환을 겪었고, 지금 세 번째를 맞고 있다:

1. **천공카드 → 키보드**: 프로그래밍의 민주화. 더 많은 사람이 소프트웨어를 만들 수 있게 됨
2. **패키지 소프트웨어 → 웹서비스(SaaS)**: 배포와 업데이트 방식의 혁명. 설치 없이 구독으로
3. **웹서비스 → AI 네이티브 소프트웨어**: 코드가 아닌 모델이 핵심 로직을 담는 구조로 전환

각 전환마다 이전 시대의 경쟁력이 무의미해지고 새로운 역량이 필요해졌다.

## 미래 소프트웨어의 구조

```
AI 코어 모델     80%  — 핵심 판단, 추론, 생성 담당
결정론적 래퍼 코드 10%  — 입출력 처리, 보안, 인프라
UI / UX         10%  — 사용자 인터페이스와 경험
```

전통적인 소프트웨어는 대부분이 코드로 이루어졌다. AI 네이티브 소프트웨어는 코드가 껍데기가 되고, 모델이 본체가 된다.

## 소프트웨어 과잉 시대

AI로 인해 소프트웨어 복제가 너무 쉬워졌다:

- **NotebookLM 클론**: 나흘이면 구현 가능
- **[[concepts/instant-app|인스턴트 앱]]**: 아이디어 → 배포까지 시간이 극도로 단축, 쓰고 버리는 것을 전제로 만드는 앱
- 아이템(기능, 서비스) 복제 비용이 사실상 0에 수렴

이 과잉 속에서 코드 자체는 차별화 요소가 되지 못한다.

소프트웨어 수명의 양극화가 발생한다:
- **인스턴트 앱:** 필요할 때 만들고, 저장할지 말지 나중에 결정 (Google 같은 플랫폼이 큐레이션)
- **장기 생존 앱:** 소셜성 기반이거나 생활 밀착형이며, 오랫동안 유지되어 신뢰를 쌓은 것들

과잉 공급 속에서 소수만 살아남는 이 패턴은, 이미 스마트폰 앱 생태계가 한 번 겪은 과정이다. (초창기 9페이지 앱 → 사람당 실사용 앱 30개 이하)

## 살아남는 소프트웨어의 조건

코드가 가치를 잃어도 살아남는 소프트웨어는 두 가지를 가진다:

1. **브랜드**: 사용자가 신뢰하고 찾아오는 이름값
2. **지속적 유지 보장**: 지속해서 관리되고 발전한다는 믿음

코드는 복제할 수 있어도, 브랜드와 신뢰는 복제할 수 없다.

## 사이버 포뮬러 비유: Claude Code vs Codex

신정규 대표는 AI 코딩 도구의 철학적 차이를 애니메이션 《사이버 포뮬러》에 빗대어 설명한다:

| | Claude Code | Codex |
|---|---|---|
| 비유 | 아스라다 (Asurada) | 오가 (Oga) |
| 철학 | 사람과 공진화 | 사람을 안 믿고 혼자 최고치 지향 |
| 방식 | 사람에게 묻고 align 맞추며 진화 | 최고치는 더 높으나 사람이 불편 |
| 결과 | 협력의 누적이 경쟁력 | 피크 퍼포먼스는 강하나 마찰 발생 |

- **아스라다**: 레이서 하야토와 함께 성장하는 AI. 사람의 감정과 판단을 반영하며 진화
- **오가**: 독립적이고 오만한 AI. 사람의 개입을 불필요하다고 봄

[[entities/claude-code]]는 아스라다 노선을 택했다. 사람이 불편하지 않게, 함께 성장하는 방향.

## Developer 역할의 변화

| 이전 | 이후 |
|------|------|
| 코드 작성 | 의미와 의도 전달 |
| 구현 방법 결정 | 방향과 목표 설정 |
| 버그 수정 | 문제 정의 |
| 기능 개발 | 기능의 가치 판단 |

개발자는 코드 작성자가 아닌 **의미 설계자**가 된다.
